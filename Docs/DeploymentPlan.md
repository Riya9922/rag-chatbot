# Deployment Plan: Mutual Fund FAQ Assistant

This document outlines the deployment strategy for the Mutual Fund RAG assistant, splitting the system across GitHub Actions, Streamlit, and Vercel.

## 1. Data Ingestion & Scheduler
**Platform:** GitHub Actions
*   **Role:** Automates the execution of Phase 1 (Scraping) and Phase 2 (Indexing).
*   **Trigger:** A scheduled Cron job configured in `.github/workflows/daily_scrape.yml` (e.g., runs daily at 09:15 AM IST).
*   **Process:** 
    1. Spins up an Ubuntu runner.
    2. Installs Python dependencies.
    3. Runs `python run_scheduler.py --once`.
    4. Persists the updated ChromaDB `vector_db` and JSON files.
*   **Artifacts:** The updated vector database can be committed back to the repository or pushed to an external cloud storage (e.g., AWS S3) for the backend to consume.

## 2. Backend & RAG Inference
**Platform:** Streamlit (via Streamlit Community Cloud or Containerized)
*   **Role:** Acts as the core RAG inference engine and backend logic handler.
*   **Process:**
    1. Pulls the latest vector database generated by GitHub Actions.
    2. Initializes the Embedding model and LLM integration.
    3. Exposes the endpoints or interaction logic for the user queries.
    4. Handles the "Refusal Detector" logic to block advisory queries.
*   **Note:** While Streamlit is traditionally a frontend framework, it will be utilized here to encapsulate the backend RAG logic. If exposing REST endpoints to Vercel is strictly required, a lightweight FastAPI wrapper running alongside or instead of Streamlit might be necessary in the future.

## 3. Frontend User Interface
**Platform:** Vercel
*   **Role:** Hosts the lightweight, minimal UI facing the retail investors.
*   **Framework:** Next.js or React.
*   **Features:**
    *   Displays the Welcome Banner and Example Queries.
    *   Handles the chat interface.
    *   Ensures the mandatory disclaimer ("Facts-only. No investment advice.") is always visible.
    *   Parses and displays the citations and "Last updated from sources" footer appended by the backend.
*   **Integration:** Communicates with the Streamlit backend via API calls or WebSocket connections to fetch the RAG-generated answers.
